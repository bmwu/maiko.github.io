防AI淘汰指南

人工智能已不再是遥远的概念，而是真切地融入了我们的生活，深刻影响着各行各业的发展。

在这样的大背景下，如果还对AI一知半解，那就如同逆水行舟，必将被时代的浪潮淘汰。

以下 10 个核心术语，让我们深入剖析下：

1. 大预言模型（LLM，Large Language Model）

  • 定义：这玩意就是基于深度学习技术，参数规模大的吓人，达到千亿级别的生成式模型，比如 GPT-4，Claude 都是它的“代表作”。它厉害之处就在于，能跟人一样理解和生成自然语言，感觉就像给机器脑袋里装了个“语言大师”。
  • 重要性：LLM 可是 AI 产品的“心脏”，起着关键作用。我们得清楚它的能力范围，比如它的“记忆力”能记住多少上下文内容，生成的东西质量怎么样，还有在哪些场景能用（比如对话、内容生成之类的）。以及，你得知道它在什么时候是在靠谱输出，什么时候又是在“一本正经的胡说八道”。

2. Transformer 架构

  • 定义：当下几乎所有主流大模型赖以构建的“骨架”，其核心在于创新性地引入了自注意力机制。凭借这一独特机制，Transformer 架构能够精准且高效地处理各类序列数据。无论是篇幅冗长、结构复杂的学术论文，还是情节跌宕起伏的长篇小说， Transformer 架构都能轻松理解其含义，并且基于理解快速生成高质量的内容，对它来说，长文本理解和生成完全就是“小菜一碟”。
  • 重要性：要是搞懂了它的原理，比如 Encoder-Decoder 机构，那就相当于掌握了“秘密武器”。调整深入文本长度，能让交互更丝滑流畅。

3. 提示词工程（Prompt Engineering）

  • 定义：简单来说，就是通过优化输入给模型的指令（Prompt），让模型输出的东西更准确。就像给模型“打小抄”，教它怎么给出正确答案。

幸福密码，“拥有有趣的灵魂，做有趣的人”

在喧嚣纷扰的尘世中，我们常常奔波忙碌，追逐着各种目标，却在不经意间忽略了生活的本质 —— 幸福。

1、有趣的灵魂：点亮生活的火花

林语堂先生说：“幸福的生活中，一定少不了幽默。” 一个有趣的灵魂，就如同黑暗中的一束光，照亮生活的每一个角落。它能让平淡的日子变得生动有趣，让艰难的时刻充满希望与勇气。
在林语堂的笔下，我们看到了他对生活的独特观察与幽默表达，无论是对人性的洞察，还是对社会现象的调侃，都充满了智慧与趣味。
拥有有趣的灵魂，不是刻意去制造笑料，而是在面对生活的种种时，保持一颗乐观、豁达的心，用幽默的态度去化解矛盾与困境，让生活充满欢声笑语。
正如林语堂所言：“有趣的人，总能从困境中笑着走出来。” 这样的人，无论身处何地，都能成为生活的主角，用自己的光芒感染身边的人，让幸福在人群中传递。

2、简单的热爱：回归生活的本真

在这个物欲横流的时代，我们常常被各种欲望所困扰，追求着更多的财富、更高的地位、更大的名气，却忘记了生活中那些最简单、最纯粹的美好。
林语堂先生却懂得在平凡的生活中寻找乐趣，饮茶、读书、写作，这些看似普通的小事，却成为了他生活中不可或缺的一部分，也让他从中获得了无尽的满足与幸福。
他说：“幸福是一种对平凡生活的深深喜爱。” 当我们拥有一份简单的热爱时，就会发现生活中的每一个瞬间都充满了意义。一杯香茗，一本好书，一片落日余晖，都能让我们感受到生活的美好与宁静。
这种简单的热爱，让我们回归生活的本真，不再被外界的喧嚣所干扰，用心去感受生活的每一个细节，用爱去拥抱生活的每一份馈赠，从而收获真正的幸福。

3、淡然的心态：拥抱生活的自由

林语堂一生追求 “无用之美”，他不为名利所累，不被世俗所束缚，以一种淡然的心态面对生活中的种种。他认为，幸福不需要刻意去追逐，而是要学会 “随遇而安”。
当我们拥有一种淡然的心态时，就能以平和的心境去看待生活中的起起落落，用从容的态度去应对生活中的风风雨雨。正如林语堂所说：“心态放平，生活就会多一分自在。” 这样的我们，才能真正成为生活的主人，享受生活的每一刻。

4、小题大做：创造生活的惊喜

林语堂常常把生活的 “小题” 当作 “大事” 来做，这种 “小题大做” 的态度，让他的生活充满了惊喜与乐趣。
他会为一壶茶的香气而陶醉，为一本书中的精彩篇章而欢呼，为一片夕阳的美丽而驻足。
在我们看似平凡无奇的生活中，其实也蕴含着无数的小美好，只是我们常常忽略了它们的存在。
如果我们能像林语堂一样，用心去发现生活中的这些小细节，并将它们当作大事来对待，那么我们的生活也会变得丰富多彩。为了一顿美味的早餐，精心挑选食材，用心烹饪；为了一个小小的梦想，全力以赴，努力拼搏；为了与家人朋友的一次相聚，认真准备，尽情享受。
当我们把生活中的每一件小事都当作大事来做时，就会发现，幸福其实就在我们身边，触手可及。

5、平衡现实与理想：奏响生活的乐章

林语堂很聪明，他既懂得享受生活，又能应对世俗的压力。他从不被理想主义困住，而是将理想融入现实，找到了属于自己的生活节奏和步伐。
在现实生活中，我们常常面临着理想与现实的矛盾与冲突。一方面，我们有着美好的理想和憧憬，渴望实现自己的人生价值；另一方面，现实的压力却让我们不得不低头妥协。
而林语堂的人生哲学告诉我们，现实和理想并不对立，它们是幸福的两翼。我们不能只沉浸在理想的世界中，不顾现实的困境；也不能为了现实的利益，放弃自己的理想。只有将理想与现实相结合，找到它们之间的平衡点，才能让自己的生活更加充实和有意义。
就像林语堂一样，在追求文学梦想的同时，也能在现实生活中找到乐趣和满足，用智慧去平衡现实与理想，奏响属于自己的生活乐章。

6、发现美：感悟生活的幸福

林语堂说过：“最懂幸福的人，不是拥有最多的人，而是能发现最多美的人。” 在我们的生活中，美无处不在，只是需要我们用心去发现。
一片叶子的飘落，一朵花儿的绽放，一段温暖的话语，一个善意的微笑，都是生活中的美好瞬间。当我们拥有一双发现美的眼睛时，就会发现，幸福其实很简单，它就在我们身边的每一个角落。
那些能够在平凡中发现美的人，总是能感受到更多的幸福。因为他们懂得珍惜生活中的每一份美好，用心去感悟生活的点滴，用爱去温暖自己的心灵。
正如林语堂所说：“幸福不是拥有，而是发现和感知。” 让我们学会像林语堂一样，用心去发现生活中的美，用爱去感悟生活的幸福，让自己的生活充满诗意与美好。
在这个快节奏的时代，让我们停下匆忙的脚步，去拥有一个有趣的灵魂，一份简单的热爱，一种淡然的心态，学会小题大做，用智慧平衡现实与理想，用心去发现生活中的美。
相信在不久的将来，我们都能找到属于自己的幸福密码，开启一段充满诗意与美好的人生旅程。

Hexo 笔记

Pages

创建 git 的 repository，在settings中，pages 中可以设置自定义域名，比如 maikoTalks.cn。
域名需要做域名解析，解析到 github 的 地址，比如 maiko.github.io。

CNAME

问题：直接在github库中添加CNAME文件，发现每次执行hexo d之后，CNAME文件就被覆盖
回答：不能直接在库中添加CNAME文件，直接在项目的source文件夹中添加CNAME文件，和在库中添加一样你的域名，之后执行
hexo clean
hexo g
hexo d

添加README.md

问题：添加README.md后，执行hexo d后，README.md文件没有被渲染
回答：修改hexo项目的_config.yml文件，将skip_render参数的值设置上。skip_render: README.md，保存退出即可。
使用hexo d 命令就不会在渲染 README.md 这个文件。

DeepSeek 学习手册

API 搭建知识库

如果本地数据不涉密,还想获得最佳使用效果,那肯定得选满1血的DeepSeekR1模型。
我们来看看怎么利用API用上满血的R1模型(671B)。

1、先下载一个叫Cherry Studio的软件。
地址: https://cherry-ai.com/download

2、登录/注册「硅基流动」,新用户会赠送2000万Token额度。
地址: https://cloud.siliconflow.cn/i/S97XY4rE

3、来到API密钥生成界面,创建或者复制已有的密钥。

4、来到CherryStudio,配置 API Key。

5、在「硅基流动」模型广场首页,排在前两位的就是「硅基流动」和「华为云」合作发布的DeepSeek R1/V3模型。
如果需要推理能力,记得打开并复制R1模型的名称。
在「Cherry Studio」模型服务的硅基流动下方,添加相应模型。

6、现在对话模型有了R1,还缺少一个嵌入模型。

嵌入模型的主要作用是将本地文件的内容转换成有意义的数字,存储到向量数据库中。
在用户提问时,利用RAG技术在数据库中搜索到相似答案最终回复用户。
我们再配置一个向量模型:BAAI/bge-m3,如果希望搜索的精准度更高,可以选择Pro/BAAI/bge-m3。

7、在「Cherry Studio」创建知识库,选择刚才配置的嵌入模型,这样就会自动利用对应的模型来向量化数据。

8、上传本地文件进行向量化。
如果本地PDF文件是扫描件、手写件,或者带有复杂的表格和数学公式,解析效果会很差,甚至无法解析。
这样才可以正常回复知识库的内容:

如果追求执行比,推荐使用Doc2x:https://doc2x.noedgeai.ccom?inviteCode=4A6KOD
如果希望更加稳定,那么可以考虑Textin:https://www.textin.com/market/detail/pdf_to_markdown
当我们上传文件后,箭头指向的图标如图所示,则代表向量量化成功。

9、测试使用,这一步添加助手,并选择刚配置的满血R1模型。

当浓缩接触到冰水，如此迷人

平时不怎么喝咖啡，如果喝那肯定就是美式了，那是简单即美味，萃取经典浓缩咖啡，以水调和，香气浓郁蔓溢。

冰美式 Americano

热美式冷了会变酸，温度的改变会影响咖啡的酸度，所以在炎炎夏日，肯定是想来一杯沁凉畅快的冰美式。
冰美式没有热美式那么多花哨的名字，区别就在于冰的多少。

冰美式又不同于热美式，不那么苦；而且浓缩加到杯子里只跟水混合，所以随着时间会慢慢的变淡。没想到当浓缩咖啡接触到冰水，竟然如此迷人。

还有一种更“冷酷”的方式：在杯子里加满冰块，然后直接把炽热的浓缩咖啡液浇在冰块上。当冰块慢慢融化，咖啡味道也随之由浓转淡，呈现出更多细致的风味

2018 总结 - 为爱而生

18年早已过去，崭新的19年也已过了1/4，按照国际惯例，一年一度的总结过去计划未来的日子早就应该到来，如今却姗姗来迟。

18年和以往唯一不同的是，以前只是“怅然若失”的假装自己到了“而立之年”，而19年，真的是步入“而立之年”了，若说自己没有那么一丝的感伤和恐惧，那是自欺欺人，但既然“木已成舟”，也只能欣然接受了，而且这一天终究是要来的。这也可能是人年纪越大，越怕过年的原因之一吧。

工作的第8个年头，也就是32岁的时候，突然有了一种很强烈的危机感，这种危机感主要来源于年龄的增长，因为你会发现和你一起共事的同事们越来越年轻，当然这不是最要紧的，最要紧的是对工作他们可以投入更多的时间，而你不仅仅是公司的一名员工，所以在处理工作的方式和方法就需要更多的“智慧”，你必须要用更高的“能力”来弥补工作时长上的短板，就像计算机设计中，要用“空间”来换取“时间”的概念一样，所以这些危机感应该是和你的能力成反比的，而这一年为了“应付”这些危机感，加速和加强了我对软件编程的理解，但也因此牺牲了很多健身的时间也是比较遗憾的。

17年我离开LOCKet，选择来到Terminus；19年我离开Terminus，再次带着期望和问题去往一个新的环境。
这一年，很辛苦，数不清的加班，尝不尽的酸甜；这一年，因为有爱情、友情、亲情，为爱而活。

18年计划

总结

这一年，我与她迈进了婚姻的殿堂。结婚那天，她当着所有人含泪道：我爱你。很高兴遇见你，一切来的都刚刚好，我会好好爱你，往后余生，请多多指教。

这一年，带着她去了仙本那。欠你的旅行总需要兑现的。卡帕莱岛那白净的沙滩、清可见底的海水、五彩缤纷的珊瑚礁、自在悠游的鱼群海龟，宛如梦境之地～

这一年，她学了无数的厨艺，做得了烘培，总能把我吃撑，体重也chengchengcheng的往上窜，也许有一种胖就叫幸福胖吧。

这一年，我们完成了家庭旅行，深圳/香港/澳门，走过了很多地方，看过了很多风景，带来了新的体验，辛苦了我的她。

这一年，她实现了诺言，带着我来到了日本关西，暴走3天，走遍京都/神户/奈良，还n进n出药妆店，东西塞满了箱子。

这一年，第一次去外省参加了婚礼，满满的祝福送给新人。顺道完成了”打卡五岳之一”的衡山，和见识了长沙的美食和人海。相较于衡阳，长沙还是一座比较友好的城市，不过以后估计不会在节假日出游了，各种瘫痪呀。

这一年，几乎都是八点班车，深夜十点而归，工作/杂事占据了大部分的精力，似乎离学习也渐行渐远。但再忙再累，只要看到她留的便条，都会泛起嘴角，18年，有你真好！

这一年，认识了”新零售”，接触了”供应链”，探索未知发现视界，满足了我的好奇心。非常多的矛盾和挑战，自己的能力也提升了不少。感谢各位的帮助和提携，和那么多个并肩作战的日夜。

这一年，我的blog停更了，更多的贡献在了lark，成长很多，不能共享出来与君共勉了。

这一年，学习了一些新技术，业务架构，技术架构，中间件，分布式，增加了大型分布式、高并发、高负载（大数据量）、高可用性系统设计开发经验。

这一年，深深的感受了单纯技术的无力感，所以转了型，从技术到产品，做一个会技术的产品，一个会武术的流氓。

19年计划

初心易得，始终难守，耐撕前行，何必慌张，送给每一个不辜负时间的人，愿我们的每一份努力都不负期望。
